# Vercel 배포 및 데이터베이스 설정 가이드

이 가이드는 만든 단축 URL 서비스를 Vercel에 배포하고, 데이터가 사라지지 않도록 데이터베이스(Vercel KV)를 연결하는 방법을 설명합니다.

## 1. Vercel 프로젝트 생성

1.  [Vercel 대시보드](https://vercel.com/dashboard)로 이동합니다.
2.  **"Add New..."** -> **"Project"**를 클릭합니다.
3.  GitHub 레포지토리를 연결하고 `Import`를 클릭합니다.
4.  설정 화면에서 별다른 변경 없이 **"Deploy"**를 클릭합니다.
    *   *초기 배포는 성공하겠지만, 데이터베이스가 없어서 기능이 제대로 작동하지 않을 수 있습니다.*

## 2. Vercel KV (데이터베이스) 생성

1.  Vercel 대시보드 상단 메뉴에서 **"Storage"**를 클릭합니다.
2.  **"Create Database"** -> **"KV (Redis)"**를 선택합니다.
3.  데이터베이스 이름(예: `yaksu-db`)을 입력하고 지역(Region)을 선택한 뒤 **"Create"**를 클릭합니다.
    *   *지역은 `Seoul, South Korea (icn1)`이 있다면 가장 좋고, 없다면 `Tokyo (hnd1)` 등을 선택하세요.*

## 3. 프로젝트와 데이터베이스 연결

1.  생성된 KV 데이터베이스 화면에서 왼쪽 메뉴의 **"Projects"**를 클릭합니다.
2.  **"Connect Project"**를 클릭하고, 방금 만든 프로젝트(`yaksu` 등)를 선택하여 연결합니다.
3.  **"Connect"** 버튼을 누르면 환경 변수(`KV_REST_API_URL`, `KV_REST_API_TOKEN` 등)가 자동으로 프로젝트에 추가됩니다.

## 4. 재배포 (Redeploy)

환경 변수가 추가되었으므로, 이를 적용하기 위해 다시 배포해야 합니다.

1.  프로젝트 대시보드로 돌아갑니다.
2.  **"Deployments"** 탭으로 이동합니다.
3.  가장 최근 배포의 오른쪽 점 세 개(...) 메뉴를 클릭하고 **"Redeploy"**를 선택합니다.
4.  배포가 완료되면 제공된 도메인(예: `https://yaksu.vercel.app`)으로 접속하여 테스트해 봅니다.

## 5. (선택 사항) 로컬 개발 환경 설정

로컬(내 컴퓨터)에서도 실제 데이터베이스를 쓰고 싶다면:

1.  터미널에서 다음 명령어를 실행하여 Vercel 환경 변수를 가져옵니다.
    ```bash
    npx vercel env pull .env.local
    ```
2.  이제 `npm run dev`로 실행하면 로컬에서도 Vercel KV에 데이터가 저장됩니다.
    *   *이 과정을 건너뛰면 로컬에서는 임시 메모리 저장소를 사용하며, 서버를 끄면 데이터가 사라집니다.*
